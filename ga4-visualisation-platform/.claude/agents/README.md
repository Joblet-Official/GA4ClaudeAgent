# Claude agents — GA4 visualisation pipeline

Four project-level Claude Code subagents that mirror the LLM brains in `src/brains/`. Same logical jobs, different execution path: Claude reasons over the prompts inside its own context, instead of being routed to Groq / Cerebras / etc. via the OpenAI SDK.

Two parallel implementations live side-by-side intentionally:

- **The LLM brains** (`src/brains/*.ts`) run in production via the Next.js routes — fast, cheap, predictable.
- **The Claude agents** (this directory) are useful for: prompt design iteration, deep debugging of a single stage, A/B comparison against the production brain, and the parts of the pipeline where Claude's longer context is genuinely helpful.

## Agents

| # | Slug | Job | Tools | Mirrors |
|---|---|---|---|---|
| 1 | `ga4-intent` | Classify question into Intent JSON (`report_type`, `sub_questions`, `scope`, `ambiguity_flags`) | — | `src/brains/brain1_intent.ts` |
| 2 | `ga4-metrics` | Build GA4 Data API query specs, catalog-grounded | Read | `src/brains/brain2_metrics.ts` |
| 3 | `ga4-gaps` | Decide `approved` / `default_applied` / `needs_clarification` | — | `src/brains/brain3_gaps.ts` |
| 4 | `ga4-data-access` | Catalog validation + Tool Layer execution. Returns raw GA4 rows | Read, Bash | `src/support/tools/runGA4Query.ts` |

## Invoking from Claude Code

```
Agent({
  subagent_type: "ga4-intent",
  description: "Classify question",
  prompt: "question: sessions by country last week\nmemory: null\ntoday: 2026-05-20\n\nReturn ONLY the JSON object."
})
```

Each agent returns its single JSON output. Chain by feeding one agent's output into the next agent's prompt — the main Claude loop is the orchestrator.

## Catalogue

`catalog/ga4_catalog.json` is the single source of truth for valid GA4 field names. Generated by `scripts/refresh_catalog.py` from:
- GA4 Data API metadata for property `516147906` (378 dimensions + 113 metrics)
- `../gtm_snapshot.latest.json` for joblet-specific event names (10 literal events from GTM tags)

Agents 2 and 4 read it directly via the Read tool. Agent 4 also re-validates every Brain 2 output before executing — defence in depth.

## What's NOT a Claude agent

Brains 5 and 6 (Data Handling and Visualisation) are pending. When built, they'll get their own files here.

The Tool Layer itself (`runGA4Query`) is pure TypeScript code, not an agent. Agent 4 *invokes* it via Bash rather than reimplementing it.

## Rules carried over from the plan

- **Never modify GTM.** Read-only via `pull_gtm_tags.py`.
- **Never invent GA4 field names.** Catalog grounding is enforced by Agent 4's validation step.
- **Never interpret the data.** Agents 1-4 are setup + execution. Interpretation belongs to Brain 5/6 (when built) — and even those don't editorialise; they just structure for the viewer.
